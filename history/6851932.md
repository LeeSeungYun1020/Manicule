# 6851932 — refactor(core:data): N+1 최적화를 위해 BookEntryWithCurrentPage에 @Relation 적용

**Files**: 
- `LibraryRepositoryImpl.kt`
- `BookEntryDao.kt`
- `BookEntryWithCurrentPage.kt`

## 결정
- `LibraryRepository`의 기존 `async/awaitAll` 병렬 처리 방식을 제거하고, Room의 `@Relation`을 사용하여 `BookEntryWithCurrentPage`에서 관련된 책 정보를 한 번에 가져오도록 수정

## 이유
- 앱 레벨에서 코루틴을 통해 다건을 병렬로 조회하는 것보다, DB 레벨(Room `@Relation`)에서 한 번에 쿼리하여 데이터를 조립하는 것이 성능 및 메모리 측면에서 효율적임
- `Repository` 계층의 코드가 단순화되며 불필요한 비동기 컨텍스트(coroutineScope) 생성을 방지함
