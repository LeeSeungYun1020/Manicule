# 80e7610 — fix: NlkApi에서 중복되는 cert_key 파라미터 제거

**Files**: NlkApi.kt, NlkApiTest.kt

## 결정
- NlkApi.kt의 searchBooks 메서드에서 cert_key 필수 파라미터를 제거함.

## 이유
- NlkAuthInterceptor에서 이미 BuildConfig.NLK_AUTH_KEY 값을 cert_key 쿼리로 자동 주입하고 있음. 
- API 인터페이스에 cert_key 파라미터가 필수값으로 남아있으면, 호출부에서 불필요하게 키를 전달해야 하므로 인터셉터의 목적과 맞지 않음.
