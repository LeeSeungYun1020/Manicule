# 4c28b94 — Refactor: Change ReadingRecordDao.upsert to use @Upsert

**Files**: `core/database/src/main/kotlin/com/leeseungyun1020/manicule/core/database/dao/ReadingRecordDao.kt`, `core/database/src/androidTest/kotlin/com/leeseungyun1020/manicule/core/database/dao/ReadingRecordDaoTest.kt`

## 결정
- `ReadingRecordDao.upsert`에 `@Insert(onConflict = REPLACE)` 대신 `@Upsert`를 사용하도록 수정.

## 이유
- 기존의 `REPLACE` 전략은 중복 삽입 시 기존 레코드를 삭제 후 재삽입하여 PK(`id`)가 계속 변경되는 문제가 있었음.
- `id` 변경으로 인한 UI의 깜빡임 등을 방지하고, "최신 데이터로 교체한다"는 본래 의도에 맞게 기존 레코드의 값을 진짜로 업데이트(`UPDATE`)하여 `id`를 유지하는 `@Upsert`가 더 적합하다고 판단함.
