# 29c8161 — fix: ReadingRecordDao.upsert가 UNIQUE 제약 조건 하에서 업데이트되도록 수정

**Files**: `core/database/src/main/kotlin/com/leeseungyun1020/manicule/core/database/dao/ReadingRecordDao.kt`

## 결정
- `ReadingRecordDao.upsert` 내부에서 `isbn`과 `date`를 이용해 기존 레코드의 `id`를 먼저 조회하고, 존재할 경우 해당 `id`를 포함하여 `@Upsert`를 호출하도록 변경.

## 이유
- Room의 `@Upsert`는 Primary Key(`id`) 충돌 시에만 `UPDATE`로 동작함.
- `id`가 자동 생성(`id=0`)인 상태에서 UNIQUE 인덱스(`isbn`, `date`) 충돌이 발생하면, Room은 PK 충돌이 아니므로 업데이트를 수행하지 않고 조용히 삽입을 무시(또는 실패)함.
- PK를 유지하면서 최신 데이터로 업데이트하려는 의도(`@Upsert` 전환 목적)를 달성하기 위해, 기존 레코드의 `id`를 수동으로 주입하는 방식으로 우회하여 해결함.
