# 036d0f7 — ai: ktlint(formatter) + detekt(analysis) 적용

**Files**: `.editorconfig`, `config/detekt/detekt.yml`, `AndroidLintConventionPlugin.kt`

## 결정
ktlint = 포맷 / 네이밍, detekt = 복잡도 / 잠재 버그. 역할 분리. `manicule.android.lint` 로 모든 모듈 자동 적용.

## 이유
- 둘 다 style 룰 보유 → 동시 활성 시 권고 충돌 (예: trailing comma).
- ktlint `ktlintFormat` 자동수정 강력 → 포맷 / 네이밍 / import 일임.
- detekt 의 cyclomatic complexity / dead code / null deref 영역만 활성. `detekt-formatting` 은 ktlint 와 100% 중복이라 미사용.
- 신규 룰 추가 시: 포맷 / 스타일 → `.editorconfig`, 그 외 → `detekt.yml`.
- 위반 시: `ktlintFormat` → 코드 수정 → 정말 안 되면 `@Suppress("DetektRule")`.
