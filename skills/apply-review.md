# 코드 리뷰 반영

## 전제 조건

- 사용자가 요청한 PR이 열려 있고(open), 리뷰가 작성되어 있음.
- `LeeSeungYun1020` 계정으로 전환 가능(`gh auth status`, `gh auth switch`)

## 실행 절차

리뷰 분석 -> 리뷰 수정 -> 답글 생성 -> 답글 작성 순서로 진행.

### 리뷰 분석

1. PR 상세 내용 및 변경 사항 확인.
2. PR 브랜치를 로컬로 패치하여 체크아웃. 코드 직접 확인.
3. PR 내 리뷰 내용에 대한 타당성 검토 진행. 수정 필요성 확인.
4. 수정이 필요한 리뷰는 수정 방향(대안) 정리.
5. 사용자에게 리뷰 리스트를 표시. 각 리뷰는 번호로 구분 가능해야 함.

### 리뷰 수정

1. git branch checkout
2. 한 번에 하나의 리뷰 내용에 대한 수정 진행
3. 사용자에게 리뷰 내용 표시. 사용자가 질문하는 경우 문답 진행.
4. 사용자가 수정을 지시하면 해당 리뷰에 대해 코드 `code-change.md` 파일 실행 단계 수행.
5. 모든 리뷰 수정이 완료되면 코드 수정 파일 `code-change.md`에서 검증 단계 수행.
6. git push

### 답글 생성

1. 분석 및 수정 내용 바탕으로 각 리뷰마다 수정 내용 또는 수정하지 않는 이유를 간결히 작성.
2. 사용자에 표시. 수정 대기.

### 답글 작성

1. `LeeSeungYun1020` 계정으로 전환(`gh auth status`, `gh auth switch`)
2. PR에 작성된 코멘트 ID 목록을 조회(`gh api /repos/:owner/:repo/pulls/<PR번호>/comments`)
3. 각 리뷰마다 작성한 답글 게시(`gh api -X POST /repos/:owner/:repo/pulls/comments/<조회한_comment_id>/replies`). 별도 comment로 작성되지 않도록 유의.

## 제약 사항

- PR에서 전체 리뷰를 가져와 순서대로 번호를 매김.
- 필요하지 않은 부차적인 내용(인사말)과 미사여구 삭제.
- 간결하고 명확하게 수정 내용, 수정하지 않은 이유에 대해 작성.
- 에러 방지를 위해 gh 명령어를 &&로 묶어 실행하지 말고, 단계별로 분리하여 안전하게 순차 실행.

## 예외(에러) 대응

- 계정 변경, 권한 문제, 답글 전송에 실패하면 중단하고 사용자에 표시.
- 각 진행 중인 단계 표시(예: `<APPLY-리뷰 분석-5>`, `<APPLY-답글 생성-2>`)

## 최종 완료 및 검증

- github PR 상태 조회(`gh pr status`)하여 코멘트에 답글이 정상 등록되었는지 확인.
